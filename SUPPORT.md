# Support

Use GitHub Discussions for setup and workflow questions. Use Issues for reproducible bugs and focused feature requests. Security vulnerabilities and leaked credentials must be reported privately as described in `SECURITY.md`.

Before asking for help, run:

```bash
python run.py --db data/release-check.db release-check
```

Share synthetic or publicly redistributable fixtures only. Never attach confidential procurement files, account lists, credentials, or personal contact data.
